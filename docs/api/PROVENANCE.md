## Overview

This document defines the provenance tracking and reproducibility architecture for the Adaptive Textbook Helper, using W3C PROV-DM (PROV Data Model) as the foundational standard.

### Goals

1. **Complete Lineage**: Track every artifact from origin to consumption
2. **Reproducibility**: Enable exact re-creation of any processing run
3. **Auditability**: Support forensic analysis and scientific evaluation
4. **Counterfactual Replay**: Re-simulate with alternative policies

---

## PROV-DM Model

### Core Concepts

The PROV-DM model defines three primary constructs:

| Construct | Description | Example |
|-----------|-------------|---------|
| **Entity** | Thing that exists | PDF file, chunk, concept node |
| **Activity** | Process that changes entities | OCR, chunking, LLM generation |
| **Agent** | Responsible for activity | System, human reviewer, LLM |

### Relations

```
wasGeneratedBy(entity, activity, time)
  - Chunk was generated by chunking activity

used(activity, entity)
  - Chunking activity used raw page text

wasAttributedTo(entity, agent)
  - Concept map attributed to human reviewer

wasAssociatedWith(activity, agent)
  - Generation activity associated with Kimi LLM

wasDerivedFrom(derived_entity, source_entity)
  - Explanation derived from chunks
```

## Counterfactual Replay

### Replay Architecture

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    COUNTERFACTUAL REPLAY SYSTEM                              │
└─────────────────────────────────────────────────────────────────────────────┘

  Original Run                    Replay Run
  ────────────                    ──────────

  ┌──────────────┐               ┌──────────────┐
  │  Trace Log   │               │  Trace Log   │
  │  (Fixed)     │──────────────▶│  (Fixed)     │
  └──────────────┘               └──────────────┘
         │                              │
         │                              │
         ▼                              ▼
  ┌──────────────┐               ┌──────────────┐
  │  Policy A    │               │  Policy B    │
  │  (Original)  │               │  (Replay)    │
  └──────────────┘               └──────────────┘
         │                              │
         ▼                              ▼
  ┌──────────────┐               ┌──────────────┐
  │  Decisions   │               │  Decisions'  │
  │  D1, D2, D3  │               │  D1', D2', D3'│
  └──────────────┘               └──────────────┘
         │                              │
         ▼                              ▼
  ┌──────────────┐               ┌──────────────┐
  │  Metrics     │               │  Metrics'    │
  │  HDI, CSI    │    COMPARE    │  HDI', CSI'  │
  └──────────────┘◄──────────────┴──────────────┘
```

### Replay Invariants

1. **Learner actions are fixed**: Same submissions, time gaps, hint requests
2. **Only system decisions vary**: Escalation timing, explanation availability
3. **Comparable metrics**: Same calculation methods on both runs

## Validation

### Provenance Completeness Checks

```python
def validate_provenance_completeness(entity_id: str) -> ValidationReport:
    """
    Check that every entity has:
    1. Generation activity or derivation source
    2. Attribution agent (if applicable)
    3. Complete activity chain to root inputs
    """
    report = ValidationReport()
    
    entity = load_entity(entity_id)
    
    # Check generation
    if not entity.generation and not entity.derivation:
        report.add_error(f"Entity {entity_id} has no generation or derivation")
    
    # Check attribution for human-reviewed content
    if entity.type in ["Concept", "PrerequisiteEdge"]:
        if not entity.attribution:
            report.add_warning(f"Entity {entity_id} should have attribution")
    
    # Recursively validate upstream
    for used_entity in entity.derivation.used:
        report.merge(validate_provenance_completeness(used_entity))
    
    return report
```
